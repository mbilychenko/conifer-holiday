## 4. Evaluation metrics

Each clustering was evaluated on four metric families:

- **Internal validity** (no ground truth):
  - Silhouette coefficient (subsample stability): mean of 10 independent subsamples of 5000 polygons (seed=42); reported as mean ± subsample percentile interval (2.5th–97.5th percentile across the 10 draws). A single draw can deviate ±0.02–0.05; the mean stabilises the estimate. Range [-1, 1], higher = better.
  - Davies-Bouldin index (lower = better)
  - Calinski-Harabasz index (higher = better)
- **Spatial validity:**
  - Black-Black join-count ratio under the prox_2500m weight matrix (fraction of graph edges connecting polygons with the same cluster label; higher = stronger spatial coherence). Note: Moran's I is invalid for nominal cluster labels — relabelling changes the statistic without changing the clustering. Join-count statistics are the correct test for categorical variables.
  - Intra-cluster / inter-cluster centroid-distance ratio (lower = better separation)
- **Entity validation** against 22 reference forests:
  - Detection rate (% with a cluster intersecting a 5000m buffer)
  - Single-cluster rate (% captured as exactly one cluster)
  - Tight placement rate (% with nearest cluster centroid ≤ 5 km from forest centre)
  - Area accuracy rate (% within 0.3×–3× of Wikipedia area)
- **Stability:** bootstrap ARI (full-data clustering vs. clustering on 80% subsample, 50 iterations) for the best config per algorithm.

Composite quality score (weights from `config.py`):
`composite = 0.3 × silhouette_norm + 0.2 × (1 / (1 + DB)) + 0.3 × detection_rate + 0.2 × single_cluster_rate`

where `silhouette_norm = max(silhouette, 0)` — negative silhouette values (indicating misclustering) receive zero contribution rather than partial credit.

Only configurations producing **between 80 and 400** clusters were considered eligible (the destination-density target derived from prior NFI work).

## 5. Statistical comparison

**Original design.** Friedman test across the four graph variants (treated as repeated measures) for graph-dependent algorithms (Agglomerative, SKATER, Max-P, Louvain), with Nemenyi post-hoc when significant (α = 0.05).

**Design limitation (acknowledged).** After the `spopt 0.7.0` implementation-limit exclusions (see §3) and the memory constraint that restricts Agglomerative + connectivity to `prox_2500m` only (see `config.SKIP_COMBINATIONS`), Louvain is the only algorithm with full coverage across all four graph variants. The Friedman test therefore reduces to k = 1 algorithm across n = 4 graphs, which is undefined — the test cannot produce a valid pairwise comparison under the current design.

**Future work.** Replace the Friedman design with a paired-bootstrap comparison of composite scores across the best-per-algorithm configs (re-using the bootstrap iterations from §6.2 to construct composite CIs), or restructure the experiment as a graph-construction ablation within Louvain (the only algorithm with full coverage). The cross-algorithm comparison in this paper therefore rests on the composite ranking in §6.1 and the bootstrap stability comparison in §6.2.

### 6.5 Negative result: spectral clustering

Spectral clustering (best config: 300 clusters on prox_10000m, silhouette = 0.185, DB = 0.957) achieved 100% detection rate but exhibited critically low bootstrap stability (ARI = 0.237 [0.134–0.364], n=50 iterations). This instability arises because spectral clustering on a binary adjacency matrix is sensitive to small perturbations in the graph structure — removing 20% of polygons changes the graph topology enough that the Laplacian eigenvectors rotate arbitrarily between runs, and cluster assignments do not correspond across bootstrap samples. Spectral clustering is therefore not recommended for this application; the method is included to document this failure mode.
